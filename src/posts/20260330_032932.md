# Google elimina la apertura de Android y se acerca al modelo de Apple

**Google** ha iniciado una transición técnica para restringir la instalación de archivos **APK** y limitar el acceso al código fuente de **Android**. Esta decisión reduce la brecha de apertura que históricamente diferenciaba al sistema operativo del ecosistema cerrado de **Apple**.

## El fin del 'sideloading' y el control de aplicaciones

La libertad de instalar software desde cualquier fuente, conocida como **sideloading**, enfrenta su mayor restricción con el protocolo **Advanced Flow**. Bajo este esquema, si un usuario intenta instalar una aplicación de un desarrollador no verificado, el sistema forzará un **reinicio del dispositivo** y un tiempo de espera de **24 horas**.

* **Google** busca centralizar la distribución de software para argumentar una mayor **seguridad** del usuario.
* El código base de Android (**AOSP**) dejará de ser público en tiempo real, liberándose solo **dos veces al año**.
* Desarrolladores de sistemas independientes como **LineageOS** y **GrapheneOS** verán obstaculizado su flujo de actualizaciones.

## Impacto en la lealtad del usuario y el mercado

La transformación de **Android** en un "jardín vallado" elimina el principal argumento de venta para los entusiastas de la **personalización** y el software libre. Al perder esta ventaja competitiva, la barrera de entrada para migrar al ecosistema de **iPhone** se vuelve casi inexistente para el consumidor promedio.

Esta centralización del software no solo afecta la libertad del propietario sobre su hardware, sino que otorga a **Google** un control absoluto sobre qué aplicaciones pueden ejecutarse. Con el próximo lanzamiento de **Android 17**, estas políticas se consolidarán, dejando a las **ROMs personalizadas** en una situación crítica de soporte técnico.
